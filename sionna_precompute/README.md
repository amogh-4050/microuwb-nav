# sionna_precompute

Offline Sionna RT preprocessing for the MicroUWB-Nav project.
Not part of the ROS2 runtime — runs once to produce lookup tables.

## Environment

```bash
source ~/anaconda3/bin/activate
conda activate sionna_env
```

Sionna 1.2.1 with Mitsuba 3 / Dr.JIT backend.
Do not upgrade Sionna without re-testing the full precompute pipeline.

## Launch Jupyter

```bash
cd ~/microuwb_nav_ws/sionna_precompute
source ~/anaconda3/bin/activate && conda activate sionna_env
jupyter notebook
```

## Notebooks

| Notebook | Purpose |
|---|---|
| `01_scene_sanity.ipynb` | Load room.xml, assert bounding box (0,0,0)→(5,4,3), render visual previews |
| `02_single_point_test.ipynb` | Place anchor_0 as TX, room centre as RX, compute CIR, assert LOS range ≈ Euclidean distance to <1 cm |

## What 3a-i validates

- Mitsuba scene geometry is correct (normals inward, coordinate frame matches Gazebo)
- ITU concrete material registered
- PathSolver produces a valid LOS path whose delay × c = Euclidean distance
- Anchor positions loaded from the same YAML used by the ROS anchor_publisher

## What 3a-ii will add

- Grid sweep: all 5 anchors × N×M×K grid points inside the room
- Save (anchor_id, x, y, z, range_m, delay_ns, num_paths) to HDF5 in `data/`
- Optional: furniture geometry added to scene/room.xml

## Directory layout

```
sionna_precompute/
├── notebooks/          Jupyter notebooks
├── scripts/            (populated in 3a-ii) headless grid sweep scripts
├── scene/
│   ├── room.xml        Mitsuba 3 scene — 6-surface room, ITU concrete
│   ├── meshes/         ASCII PLY mesh files (generated by gen_meshes.py)
│   └── gen_meshes.py   Regenerate meshes if geometry changes
└── data/               (populated in 3a-ii) HDF5 lookup tables
```
