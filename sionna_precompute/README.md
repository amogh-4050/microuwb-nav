# sionna_precompute

Offline Sionna RT preprocessing for the MicroUWB-Nav project.
Not part of the ROS2 runtime — runs once to produce lookup tables.

## Environment

```bash
source ~/anaconda3/bin/activate
conda activate sionna_env
```

Sionna 1.2.1 with Mitsuba 3 / Dr.JIT backend.
Do not upgrade Sionna without re-testing the full precompute pipeline.

## Launch Jupyter

```bash
cd ~/microuwb_nav_ws/sionna_precompute
source ~/anaconda3/bin/activate && conda activate sionna_env
jupyter notebook
```

| Notebook | Purpose |

## Grid sweep (step 3a-ii)

Always validate with `--dry-run` first:

```bash
cd ~/microuwb_nav_ws/sionna_precompute
conda activate sionna_env

# Validate script on 5×5×5 sub-grid (~1 min):
python scripts/precompute_cir_table.py --dry-run

# Full sweep (93×73×57 grid, 5 anchors, ~5-30 min on RTX 4060):
python scripts/precompute_cir_table.py

# Validate output:
python scripts/validate_table.py
```

If OOM on the 4060, halve chunk size:
```bash
python scripts/precompute_cir_table.py --chunk-size 5000
```

Output: `data/room_cir_table_full.h5` (~30–80 MB gzip-4 compressed).
Validation plots: `data/validation/`.

## HDF5 schema

```
room_cir_table_full.h5
├── attrs: sionna_version, frequency_hz, max_depth, generation_timestamp,
│         grid_origin, grid_resolution, grid_shape
├── /anchors        (5, 3) float32 — anchor XYZ positions
├── /anchor_ids     (5,) uint8
└── /per_anchor/
    ├── anchor_0/
    │   ├── range_m             (93, 73, 57) float32 — first-path range
    │   ├── los                 (93, 73, 57) bool    — LOS flag
    │   ├── first_path_power_db (93, 73, 57) float32
    │   ├── total_rx_power_db   (93, 73, 57) float32
    │   └── variance_m2         (93, 73, 57) float32 — Kalman measurement noise
    └── anchor_1/ … anchor_4/
```

`variance_m2` is a power-based heuristic (0.001–0.05 m² for valid cells, 0.25 m²
for no-path cells). Requires absolute calibration in step 5 Kalman filter.

## What 3a-i validated

- Mitsuba scene geometry is correct (normals inward, coordinate frame matches Gazebo)
- ITU concrete material registered
- PathSolver produces a valid LOS path: delay × c = Euclidean distance (error 0.24 cm)
- Anchor positions loaded from the same YAML used by the ROS anchor_publisher

## What 3a-ii adds

- Furniture AABB bounding boxes in scene (ITU wood: bookshelf, cafe_table, cabinet)
- Grid sweep: all 5 anchors × 93×73×57 = 386,793 grid points each
- LOS and NLOS path detection per cell
- HDF5 lookup table for ROS runtime interpolator (step 3b)
